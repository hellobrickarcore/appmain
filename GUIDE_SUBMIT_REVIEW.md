# Guide: Submit App for Review

## Prerequisites

- ✅ App Store listing complete
- ✅ Build selected and ready
- ✅ All required fields filled
- ✅ Status: "Ready to Submit"

## Step-by-Step Instructions

### 1. Final Review

**Before submitting, verify:**

1. **App Information**
   - ✅ Subtitle correct
   - ✅ Description accurate
   - ✅ Keywords relevant
   - ✅ URLs working

2. **Screenshots**
   - ✅ All device sizes uploaded
   - ✅ Screenshots show actual app
   - ✅ No personal information visible

3. **Build**
   - ✅ Correct build selected
   - ✅ Version matches listing
   - ✅ Build status: "Ready to Submit"

4. **Privacy**
   - ✅ Privacy policy URL accessible
   - ✅ Privacy nutrition labels complete
   - ✅ Data collection disclosed

### 2. Export Compliance

**Answer questions:**

1. **"Does your app use encryption?"**
   - Usually: **Yes** (HTTPS uses encryption)
   - Or: **No** (if no network features)

2. **"Does your app use, contain, or incorporate cryptography?"**
   - Usually: **Yes** (standard encryption)
   - Select: **"My app uses standard encryption"**

3. **"Does your app use, contain, or incorporate cryptography that is exempt?"**
   - Usually: **No**
   - Only if using custom encryption

**Note:** Most apps use standard encryption (HTTPS), which is exempt from export restrictions.

### 3. Content Rights

**Confirm:**

1. ✅ **"I confirm I have the rights to use all content in this app"**
   - Must check this box
   - App will be rejected if you don't own content

2. ✅ **Age rating is accurate**
   - Based on content questionnaire
   - Usually 4+ or 9+ for HelloBrick

### 4. Advertising Identifier (IDFA)

**If not using ads:**

1. Select **"No"**
   - App doesn't use advertising identifier
   - Simplest option

**If using analytics:**

1. Select **"Yes"**
2. Complete IDFA questionnaire
3. Declare usage purposes

### 5. Submit for Review

1. **Scroll to top** of App Store tab
2. Click **"Submit for Review"** button (blue, top right)
3. **Confirm submission**
   - Review warning appears
   - Click **"Submit"** to confirm

### 6. Submission Confirmation

**After clicking Submit:**

1. **Status changes** to "Waiting for Review"
2. **Email sent** confirming submission
3. **Can't edit** most fields while in review
4. **Can withdraw** if needed

## Review Timeline

**Typical timeline:**

- **Submission**: Immediate
- **In Review**: 24-48 hours (can be up to 7 days)
- **Approval**: App goes live within 24 hours
- **Rejection**: Fix issues and resubmit

**First submission** may take longer.

## During Review

**What Apple checks:**

1. ✅ App follows App Store guidelines
2. ✅ No crashes or bugs
3. ✅ Privacy policy accessible
4. ✅ Screenshots match app
5. ✅ No prohibited content
6. ✅ Functionality works as described

**You can:**

- ✅ Check review status
- ✅ Withdraw submission (if needed)
- ✅ Respond to reviewer questions
- ❌ Can't edit most fields
- ❌ Can't upload new build (must withdraw first)

## Review Status Updates

**Statuses you'll see:**

1. **"Waiting for Review"**
   - Just submitted
   - In queue

2. **"In Review"**
   - Apple is reviewing
   - Usually 24-48 hours

3. **"Pending Developer Release"**
   - Approved!
   - Waiting for you to release

4. **"Ready for Sale"**
   - Live in App Store!
   - Available to users

5. **"Rejected"**
   - Issues found
   - Check email for details

## If Rejected

**Common rejection reasons:**

1. **Crashes or bugs**
   - Fix and resubmit

2. **Missing information**
   - Privacy policy not accessible
   - Missing screenshots

3. **Guideline violations**
   - Prohibited content
   - Misleading description

4. **Technical issues**
   - Code signing problems
   - Missing icons

**What to do:**

1. **Read rejection email** carefully
2. **Fix all issues** mentioned
3. **Respond in App Store Connect** (if questions)
4. **Resubmit** after fixes

## After Approval

**When approved:**

1. **Status**: "Pending Developer Release" or "Ready for Sale"
2. **Email notification** sent
3. **App appears in App Store** within 24 hours
4. **Monitor** for user reviews and ratings

**If "Pending Developer Release":**

1. Go to App Store tab
2. Click **"Release This Version"**
3. App goes live immediately
