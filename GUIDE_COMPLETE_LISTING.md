# Guide: Complete App Store Listing

## Prerequisites

- ✅ Build uploaded and processed in App Store Connect
- ✅ Build status: "Ready to Submit"
- ✅ Screenshots captured (see GUIDE_CAPTURE_SCREENSHOTS.md)
- ✅ Privacy policy hosted publicly
- ✅ App Store description ready (APP_STORE_DESCRIPTION.md)

## Step-by-Step Instructions

### 1. Go to App Store Tab

1. Log into App Store Connect
2. Select your app: **HelloBrick**
3. Click **"App Store"** tab (left sidebar)
4. Select version: **1.0.0** (or create new version)

### 2. Select Build

1. Scroll to **"Build"** section
2. Click **"+"** or **"Select a build before you submit your app"**
3. Select your uploaded build
   - Should show: Version 1.0.0, Build 1
   - Status: Ready to Submit
4. Click **"Done"**

### 3. App Information

**Fill in required fields:**

1. **Subtitle** (30 characters max)
   - "AI Building Brick Scanner"
   - Copy from APP_STORE_DESCRIPTION.md

2. **Description** (4000 characters max)
   - Copy full description from APP_STORE_DESCRIPTION.md
   - Can include formatting (line breaks, etc.)

3. **Keywords** (100 characters max)
   - building bricks,scanner,AI,detection,collection,organization
   - Separate with commas
   - No spaces after commas

4. **Support URL**
   - Your support page URL
   - Example: https://hellobrick.app/support
   - Must be publicly accessible

5. **Marketing URL** (Optional)
   - Your website
   - Example: https://hellobrick.app

6. **Privacy Policy URL** (Required)
   - Your hosted privacy policy
   - Example: https://hellobrick.app/privacy
   - Must be publicly accessible
   - Must be HTTPS

### 4. Upload Screenshots

**For each device size:**

1. **iPhone 6.7" Display**
   - Click **"+"** to add screenshots
   - Upload screenshots in order:
     1. Home screen
     2. Scanner screen
     3. Collection screen
     4. Quests screen
     5. Profile screen
     6. Feed screen (if applicable)
   - First screenshot is shown first
   - Can reorder by dragging

2. **iPhone 6.5" Display**
   - Repeat process
   - Upload same screenshots (resized)

3. **iPhone 5.5" Display**
   - Repeat process

4. **iPad Pro 12.9"**
   - Repeat process

5. **iPad Pro 11"**
   - Repeat process

**Tips:**
- Upload at least 3 screenshots per size
- First screenshot is most important (shown in search)
- Can add captions (optional)

### 5. App Preview (Optional but Recommended)

1. Click **"+"** to add video
2. Upload 15-30 second video
3. Show key features:
   - Scanning bricks
   - Detection overlay
   - Collection organization
4. Requirements:
   - MP4 or MOV format
   - H.264 codec
   - Device-specific resolution

### 6. App Icon

- Already set from build
- Can't change here (must rebuild)
- Should be 1024×1024 PNG

### 7. Version Information

**Fill in:**

1. **What's New in This Version**
   - "Initial release"
   - Or describe features
   - Can update for future versions

2. **Copyright**
   - "© 2024 HelloBrick"
   - Or your company name

3. **Trade Representative Contact Information** (if applicable)
   - Only if selling in certain regions

### 8. App Review Information

**Fill in:**

1. **Contact Information**
   - Your email (for reviewer questions)
   - Phone number (optional)

2. **Demo Account** (if app requires login)
   - Username and password
   - For reviewer testing

3. **Notes** (optional)
   - Additional info for reviewers
   - Test instructions
   - Special considerations

### 9. Version Release

**Choose release option:**

1. **Automatically release this version**
   - App goes live immediately after approval

2. **Manually release this version**
   - You control when to release
   - Can schedule release date

3. **Release this version to specific countries**
   - Gradual rollout
   - Test in select markets first

### 10. Review and Save

1. **Review all information**
   - Check for typos
   - Verify URLs work
   - Ensure screenshots are correct

2. **Click "Save"** (top right)
   - Saves draft
   - Can edit later

3. **Status shows**: "Ready to Submit"
   - All required fields filled
   - Build selected
   - Ready for review

## Next Steps

After completing listing:
1. Review everything one more time
2. Submit for review (see GUIDE_SUBMIT_REVIEW.md)
3. Wait for Apple review (24-48 hours)

## Important Notes

- **Can edit** before submission
- **Can't change build** after submission (must create new version)
- **Screenshots** can be updated anytime
- **Description** can be updated (requires new submission for major changes)
- **Privacy Policy URL** must be accessible
- **All required fields** must be filled before submission
